# Plain - MVP仕様書
## 1. 概要
本アプリは、ターミナル上で動作するシンプルなファイルマネージャである。  
既存の高機能TUIファイラー（例: yazi等）は操作習得コストが高いため、  
本アプリは以下を重視する。

- 学習コストの低さ
- GUIファイラーに近い操作体系
- 必要最低限の機能に限定
- 迷わないUI設計

---

## 2. 対象範囲（MVPスコープ）

### 2.1 含まれる機能

- ファイル・ディレクトリの閲覧
- ディレクトリ移動
- 複数選択
- コピー / カット / ペースト
- リネーム
- 削除（ゴミ箱）
- 新規ファイル / ディレクトリ作成
- ファイル名フィルタ検索
- ソート切り替え
- パスコピー
- 外部アプリでファイルを開く
- カレントディレクトリでターミナル起動

### 2.2 含まれない機能（非対象）

- ファイル内容プレビュー
- ファイル編集機能
- Git連携
- 圧縮ファイル操作
- タブ機能
- マウス操作
- コマンド実行機能
- 高度なキーバインドカスタマイズ
- ネットワークファイル専用機能

## 3. 画面仕様

### 3.1 レイアウト
+----------------+----------------------+----------------+
| 親ディレクトリ | カレントディレクトリ | 子ディレクトリ |
+----------------+----------------------+----------------+
| |
| ステータスバー |
+----------------------------------------------------+


 
---

### 3.2 各ペインの役割

#### 左ペイン（親）
- 親ディレクトリの一覧を表示

#### 中央ペイン（メイン）
- カレントディレクトリの一覧
- 操作対象

#### 右ペイン（子）
- 中央ペインのカーソル対象がディレクトリの場合、その中身を表示
- ファイルの場合は空表示

---

### 3.3 表示項目

#### 中央ペイン
- 種別（ファイル/ディレクトリ）
- ファイル名
- サイズ
- 更新日時

#### 左・右ペイン
- ファイル名のみ（軽量表示）

---

### 3.4 中央ペイン要約 / ステータスバー

表示内容:

- ファイル数
- 選択数
- ソート状態

例:
120 items | 3 selected | sort: modified desc

下段ステータスバーは warning / error / info などの通知表示に使う。

## 4. 操作仕様

### 4.1 移動

| 操作 | 内容 |
|------|------|
| ↑ ↓ | カーソル移動 |
| ← | 親ディレクトリへ移動 |
| → | ディレクトリに入る |
| Enter | ディレクトリ: 移動 / ファイル: 既定アプリで開く |
| e | ファイルを現在のターミナル内エディタで開く |
| Backspace | 親ディレクトリへ移動 |

---

### 4.2 選択

| 操作 | 内容 |
|------|------|
| Space | 選択トグル（次行へ移動） |
| Esc | 全選択解除 |

#### 仕様
- 選択が1件以上ある場合は選択対象に対して操作
- 選択が0件の場合はカーソル対象を操作

---

### 4.3 ファイル操作

| 操作 | 内容 |
|------|------|
| Ctrl+C | コピー |
| Ctrl+X | カット |
| Ctrl+V | ペースト |
| : | コマンドパレット |
| F2 | リネーム |
| Delete | ゴミ箱へ移動 |
| コマンドパレット -> Create file | 新規ファイル |
| コマンドパレット -> Create directory | 新規ディレクトリ |

---

### 4.4 検索（フィルタ）

| 操作 | 内容 |
|------|------|
| / | フィルタ入力開始 |

#### 挙動
- 入力中はリアルタイムで絞り込み
- `Enter` で確定
- `Esc` で解除

---

### 4.5 ソート

以下を切り替え可能:

- 名前順
- 更新日時順
- サイズ順

オプション:
- ディレクトリ優先 ON/OFF

---

### 4.6 その他操作

| 操作 | 内容 |
|------|------|
| : | コマンドパレットを開き、隠しファイル表示切替を実行 |
| F5 | 再読み込み |
| Ctrl+Shift+C | パスコピー |
| Alt+Left | 戻る |
| Alt+Right | 進む |
| t | ターミナル起動（カレントディレクトリ） |

---

## 5. ファイル操作仕様

### 5.1 コピー / 移動

- 同一ディレクトリ内でも実行可能
- 上書き時の挙動:
  - 上書き
  - スキップ
  - リネーム
  → ダイアログで選択

---

### 5.2 削除

- デフォルトはゴミ箱へ移動
- 失敗時はエラーメッセージ表示

---

### 5.3 リネーム

- 単一ファイルのみ対応（MVP）
- 即時反映

---

## 6. 外部連携

### 6.1 ファイルオープン

- OSの既定アプリで開く
- OSごとの実装:
  - Linux: `xdg-open`
  - macOS: `open`
  - Windows: `start`

---

### 6.2 ターミナル起動

- カレントディレクトリで新規ターミナル起動
- OSごとに適切なコマンドを使用

---

## 7. パフォーマンス要件

- 1000件以上のファイルでも操作可能
- スクロールは遅延読み込み対応
- 子ペインは必要時のみ更新

---

## 8. エラーハンドリング

対象:

- 権限不足
- ファイル不存在
- コピー失敗
- 削除失敗

対応:

- ステータスバーまたはポップアップで表示
- アプリは終了しない

## 11. 設計方針

- 機能追加より操作の分かりやすさを優先
- 状態遷移を極力少なくする
- モーダルUIは最小限
- ユーザーが迷わないことを最優先

---

## 12. 実装方針（概要）

### 12.1 アーキテクチャ方針

- UI とロジックを分離する
- 状態は単一の構造で管理する
- 操作は Action として扱う
- 状態更新は一箇所に集約する
- ファイル操作は副作用として分離する

基本フロー:
ユーザー入力 → 状態更新 → UI再描画 → 副作用実行


---

### 12.2 UI構成方針

- Textual を使用
- 3ペイン + ステータスバーを基本構成とする
- UIは表示に専念し、ロジックを持たせない

---

### 12.3 状態管理方針

単一の状態（AppState）で管理する。

主な要素:

- 現在パス
- ファイル一覧
- カーソル位置
- 選択状態
- ソート・フィルタ条件
- クリップボード状態
- UIモード

---

### 12.4 UIモード管理

以下のモードで入力を切り替える:

- 通常閲覧（BROWSING）
- 入力中（FILTER / RENAME / CREATE）
- 確認ダイアログ（CONFIRM）
- 実行中（BUSY）

---

### 12.5 副作用方針

以下は状態更新と分離:

- ファイル操作
- ディレクトリ読み込み
- 外部アプリ起動
- ターミナル起動

結果は再度状態に反映する。

---

### 12.6 非同期処理

以下は非同期で実行:

- ディレクトリ読み込み
- ファイル操作

目的:
- UIフリーズ防止

---

### 12.7 データ管理

- 元データと表示データを分離する
- 選択はパス単位で管理する

---

### 12.8 OS依存処理

以下は専用処理として分離:

- ファイル操作
- ゴミ箱処理
- アプリ起動
- ターミナル起動

---

### 12.9 実装優先順位

1. 画面構築
2. ディレクトリ表示
3. 移動・選択
4. ファイル操作
5. 検索
6. ソート

---

### 12.10 原則

- 状態は一元管理
- UIは表示専用
- 副作用は分離
- シンプルさを最優先
