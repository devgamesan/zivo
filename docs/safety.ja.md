# 安全性について

zivo は、ファイル操作における事故を防ぐためのさまざまな安全機構を備えています。

---

## ゴミ箱への移動 (Trash)

- `d` キーまたは `Delete` キーで選択項目をゴミ箱へ移動します。
- 既定では確認ダイアログが表示されます（`config.toml` の `behavior.confirm_delete` で変更可能）。
- macOS / Linux では `send2trash` により OS 標準のゴミ箱へ移動します。
- Windows では `send2trash` 経由で Recycle Bin を使用します。

---

## 完全削除

- `D` キーまたは `Shift+Delete` で選択項目を完全に削除します。
- 完全削除は常に確認ダイアログが表示されます（`behavior.confirm_delete` の設定に関わらず確認します）。
- ゴミ箱移動とは異なり、この操作は元に戻せません。

---

## Undo（操作の取り消し）

- `z` キーで直前のファイル操作を取り消せます。
- Undo 対応操作: リネーム、貼り付け、ゴミ箱移動。
- Undo 履歴が空の場合はコマンドパレットに `Undo last file operation` は表示されません。
- Undo が可能な操作のみが Undo 履歴に記録されます。

---

## 貼り付け競合の解決

- 貼り付け先に同名ファイルが存在する場合、競合ダイアログが表示されます。
- `o`（overwrite / 上書き）、`s`（skip / スキップ）、`r`（rename / リネーム）、`Esc`（cancel / キャンセル）のいずれかを選択できます。
- 既定の動作は `config.toml` の `behavior.paste_conflict_action` で設定可能です（`prompt` / `overwrite` / `skip` / `rename`）。

---

## Symlink 操作

- ファイル操作は選択したディレクトリエントリ自体に対して行われます。
- 選択中の項目が symlink の場合も、リンク先を暗黙に辿って変更せず、symlink エントリ自体を操作します。

---

## 文字列置換のプレビュー

- 文字列置換は実行前に右ペインに diff プレビューが表示されます。
- 置換内容を確認してから `Enter` で実行するため、誤った一括置換を防げます。
- プレビュー中は `Shift+↑` / `Shift+↓` で diff をスクロールできます。

---

## アーカイブ展開時の安全措置

- 展開先に既存のディレクトリやファイルがある場合は事前に確認ダイアログを表示します。
- 展開中は status bar にエントリ件数ベースの進捗が表示されます。

---

## 外部コマンド実行

- `!` キーで 1 行シェルコマンドを実行できます。
- コマンドは zivo のプロセスとは別にバックグラウンド実行されるため、意図しない終了を防ぎます。
- 実行結果は status bar に表示されます。

---

## データ損失防止のための設計

- `config.toml` の設定値が不正でも、zivo は起動を止めません。該当項目だけ既定値へフォールバックし、初回ロード後に警告を表示します。
- `logging.enabled = true` に設定すると、起動失敗や未処理例外はログファイルに記録されるため、後から原因を調査できます。
- zivo はファイル操作を可逆的に行うことを基本設計としており、誤操作の影響を最小限に抑えます。
